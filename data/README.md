# data/

Generated by the scraper. Do not edit manually - these files are overwritten on every scrape run.

| File/Dir | Description |
|----------|-------------|
| `index.json` | Lightweight array used by the search UI. One object per series with fields: `i` (position in chunk), `t` (title), `c` (cover URL), `s` (status), `src` (source codes), `m` (max chapter), `k` (chunk number), `ua` (updated_at ISO date), `uf` (run counter when max_chapter last increased) |
| `chunks/chunk_N.json` | Full series data in 1000-entry files. Contains `sources`, `chapters`, `flame_series_id`, `demonic_id`, etc. The front-end fetches only the chunk for the selected series. |
| `scrape_state.json` | Internal scraper state. Keyed by normalised title or HTML slug. Stores `status`, `max_chapter`, `chapter_count`, `ua`, `uf` per series for incremental diff checks. |
